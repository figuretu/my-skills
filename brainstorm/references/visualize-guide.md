# Visualize Guide

脑暴过程中遇到前端布局、UI 方案等视觉内容时，按本指南生成 HTML 页面辅助用户判断。

## 适用判断

**用浏览器展示**：内容本身是视觉的
- 页面布局、组件结构、信息层级对比
- 视觉风格方向对比
- 带标注的页面/组件草图

**留在终端**：内容是文字或概念性的
- 需求澄清、scope 划分、技术选型
- 概念性 A/B/C 选择、tradeoff 讨论
- API 设计、数据建模

关于 UI 话题的问题不一定需要可视化。"导航用顶部还是侧边？"是概念问题，用终端。"这两种导航布局哪个更合适？"是视觉问题，用浏览器。

## 模板选择

### ui-compare：并排对比 2-4 个方案

```text
+--------------------------------------------------+
| 标题：这次在比较什么 / 该看什么                  |
+-------------------+------------------------------+
| 方案 A            | 方案 B                       |
| mockup            | mockup                       |
| 1-2 行说明        | 1-2 行说明                   |
+-------------------+------------------------------+
| 方案 C（可选）    | 方案 D（可选）               |
+--------------------------------------------------+
| 回到终端反馈：选 A/B/C，或说明想混合哪些部分     |
+--------------------------------------------------+
```

### annotated-mockup：单个页面/组件带标注

```text
+--------------------------------------------------+
| 标题 + 目标                                      |
+--------------------------------------------------+
| 主 mockup                                        |
|  [1] 顶部导航                                    |
|  [2] 主要操作区                                  |
|  [3] 次级信息区                                  |
+--------------------------------------------------+
| 标注说明列表                                     |
+--------------------------------------------------+
| 回到终端反馈：指出想调整的区域                   |
+--------------------------------------------------+
```

## HTML 生成规范

产出单文件 HTML，遵循：

- 内联 CSS 和必要 JS，不依赖外部 CDN
- 内容聚焦当前问题，不做无关装饰
- 页面顶部明确写出"正在比较什么"
- 方案卡片带清晰标签（A / B / C）
- 页面底部提示：请回到终端反馈

保真度：
- 默认低保真 wireframe（块状布局、占位文本）
- 用户明确要求时才提升保真度

## 输出与打开

- 输出路径：与脑暴文档同目录，文件名语义化（如 `layout-compare.html`）
- 写入后使用 `open <html-file>` 在本地浏览器打开

## 终端说明

打开浏览器后，在终端告知用户三点：
1. 浏览器里正在展示什么
2. 应该重点比较什么
3. 看完后如何在终端反馈

示例：
> 已打开本地页面，展示 3 个 dashboard 布局方向，重点看信息层级和导航密度。看完后在终端告诉我偏向 A/B/C，或说出想混合哪些部分。

## 迭代

- 用户反馈后如需调整，生成新版本 HTML（如 `layout-compare-v2.html`），不覆盖旧文件
- 下一步不需要可视化时，直接回到终端讨论
- 不读取或依赖浏览器交互日志，不要求用户在浏览器中点击
