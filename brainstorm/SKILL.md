# Brainstorm

文件驱动的结构化脑暴工具。生成脑暴文档，逐个问题引导用户讨论，结论实时回写。

## 核心边界

- 只在用户直接触发时使用，不自动触发
- 纯脑暴工具：生成问题 → 逐个过 → 记录结论，不自动衔接后续实现流程
- 文件驱动：所有问题和结论持久化到 md 文件，防止长上下文丢失前因后果

## 参考资料

`references/visualize-guide.md` 包含前端可视化展示指南。当脑暴问题涉及前端布局、UI 方案等视觉内容时，按该指南生成 HTML 页面辅助用户判断。

## 执行流程

### Step 1：理解主题与项目现状

在生成脑暴问题之前，先确保对问题背景有足够的理解。此步的提问是前置条件确认（"我们在聊什么"），不是脑暴问题本身（"我们怎么选"）。

确认上下文：
- 本次脑暴要解决什么问题
- 相关背景和约束条件
- 期望的产出（方向选择、方案对比、需求细化等）
- 如信息不足，持续提问直到上下文和用户需求足够清晰，再进入下一步

了解项目现状：
- 浏览项目大致结构（目录、关键文件）
- 根据用户的脑暴主题，找到相关代码确认当前实现状态
- 将项目现状纳入脑暴文档的「背景与上下文」段落

### Step 2：生成脑暴文档

确定输出路径：
- 用户指定路径时，使用用户指定路径
- 否则默认写入 `./.ai_docs/brainstorm/`
- 目录不存在时先创建

文件命名：`YYYY-MM-DD-<topic>.md`

文档结构：

```markdown
# 脑暴：<主题>

日期：YYYY-MM-DD

## 背景与上下文

（本次脑暴要解决什么问题，相关背景和约束）

## 问题清单

### Q1: <问题描述>
<!-- [需要可视化] -->  ← 仅当该问题涉及前端布局等视觉内容时标注
选项：
- A: xxx
- B: xxx
- C: xxx

> **结论**：（待确认）
> **理由**：

### Q2: <问题描述>
选项：
- A: xxx
- B: xxx

> **结论**：（待确认）
> **理由**：

...
```

生成初始问题列表后告知用户文档路径，开始逐个讨论。

### Step 3：逐个问题讨论

从 Q1 开始，每轮只讨论一个问题：

1. 向用户展示当前问题和选项（2-4 个选项，附简要说明和推荐理由）
2. 如该问题标注了 `[需要可视化]`，读取 `references/visualize-guide.md` 并按其规则生成 HTML 页面展示方案
3. 等待用户反馈
4. 用户确认后，将结论和理由回写到脑暴文档对应位置
5. 进入下一个问题

### Step 4：动态调整

讨论过程中，前面的结论可能影响后续问题。允许：

- 根据已有结论调整、增删后续问题
- 用户主动提出新问题时，追加到文档
- 用户要求跳过或合并问题时，灵活处理

每次调整后同步更新脑暴文档。

### Step 5：收尾

所有问题讨论完毕后：

1. 更新脑暴文档，确保所有结论已回写
2. 在文档末尾追加总结段落，概括关键决策
3. 告知用户脑暴文档最终路径

## 关键原则

- **每轮一个问题**：不要同时抛出多个问题
- **选项优先**：尽量给出 2-4 个具体选项，比开放式提问更容易推进
- **推荐先行**：每个问题给出推荐选项并说明理由，用户可以直接采纳或调整
- **文件为准**：所有结论以脑暴文档为准，终端对话是讨论过程
- **视觉辅助**：前端布局等难以文字表述的问题，自动走可视化流程
